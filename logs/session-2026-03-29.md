## Activities Section Rendering Fix

Three iterations to get Activities items rendering correctly in the dashboard.

**Root cause:** The dashboard's `cleanTripPlan()` function converts `SECTION HEADER\n────────────` into `### SECTION HEADER`. Without a blank line between the divider and the first item, the section header and items landed in the same paragraph block. The block handler sees the block starts with `<h3>` and returns it raw — items rendered as unstyled text at 16px browser default.

**Fix:**
- `·` prefix → `- ` prefix (markdown list syntax, matched by `^ *[\-\*] ` in the renderer → `<li>` elements)
- Added blank line after the `────` divider so header and items split into separate blocks
- Items now render as `<ul><li>` at 13px, matching the rest of the card
