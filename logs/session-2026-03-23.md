### 17. Dashboard (index.html) — series of display fixes
**Commits:** `e97fab7`, `b92d6a8`, `445e1c2`, `0db1ae6`
**Files:** `index.html`

Four rounds of fixes to the web dashboard:

**Opponent name parsing (`e97fab7`)**
`parseTrip()` was leaving em-dashes and `@`/`vs.` prefixes in the opponent field
(e.g. `— @ Louisiana Tech Bulldogs`). Fixed regex chain to strip `—`/`–`, leading
`vs.`/`@`/`at`, parentheticals like `(HOME)`, and trailing emoji annotations like
`★ ARMY-NAVY GAME` and `⚡ FRIDAY NIGHT`.

**Modal ASCII art cleanup (`e97fab7`)**
Added `cleanTripPlan()` to strip box-drawing characters (`═══`, `─────`, `┌─┐`,
`│...│`, `└─┘`) and convert all-caps section headers (`GAME DETAILS\n────`) to
markdown `### headings` before rendering. Added `renderStatusBlock()` to format
the `**Status: TYPE — detail | detail**` line as a styled badge with per-component
bullet points (✅/❌) instead of a run-on bold sentence.

**Key-value row formatting (`b92d6a8`)**
All plan sections (Game Details, Hotel Options, Cost Summary, etc.) were rendering
as walls of text — every `Key     : Value` pair in one `<p>` block with `<br>`
separators, same color, same font. Added kv-row detection to `renderMarkdown()`:
lines matching `  Key    : Value` (2+ leading spaces, word-char key, whitespace
before colon) are converted to `<div class="kv-row">` with key in dim 10px
monospace and value in white 13px. Added `div` to the block-element exception list
so kv blocks aren't re-wrapped in `<p>`.

**Label rename + stat split (`445e1c2`)**
"Planned" was too close to "Booked" and the header showed a confusing "0 / 12 / 13".
Renamed to "Open" (nothing confirmed yet) with "In Progress" for partially booked.
Header stats split into four distinct counts: Booked / In Progress / Open / Total.
Section group headings updated to match. Badge labels updated on cards.

**Orphaned content and note formatting (`0db1ae6`)**
Three issues in modal content:
- Box values wrapping across two `│...│` rows produced orphaned continuation lines
  (e.g. `Notes : Morning arrival — full afternoon to` + orphan `drive to Central
  Valley and settle in`). Fixed by merging continuation lines onto the preceding
  kv value in `cleanTripPlan()`.
- Option labels (`Option 1 — American nonstop...`, `Option 2 — United nonstop...`)
  rendered as unstyled plain paragraphs. Fixed by converting to `**bold**` in
  `cleanTripPlan()` so they render as distinct bold subheadings.
- Note bullets (`  - text`) had 2 leading spaces that the list regex didn't handle.
  Fixed by changing `^[\-\*] ` to `^ *[\-\*] `. Multi-line bullet continuations
  (4+ space indent) are now also merged onto the preceding bullet.
